Automata Theory

What is Finite Automata?

Finite Automata are abstract machines used to recognize patterns. They consist of a finite number of states, transitions between these states based on input symbols, and designated start and accept states.

DFA (Deterministic Finite Automaton)

In a DFA, for each state and for each symbol, there is exactly one transition to a next state. Epsilon (ε) transitions are strictly not allowed.

NFA (Non-Deterministic Finite Automaton)

An NFA can transition to multiple next states for the same symbol. It can also transition between states without consuming any input symbol, via an epsilon (ε) transition. During execution, it essentially "threads" its existence and can be in multiple states at once.

In this Simulator

  • Toggle between DFA and NFA mode at the top.
  • Add states and connect them using the sidebar.
  • Double click on states or use the sidebar buttons to mark a state as 'Start' or 'Accepting'.
  • Type a string of characters (e.g. 1011) and hit Run to watch the machine process!
